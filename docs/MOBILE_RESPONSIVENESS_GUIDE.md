# Руководство: мобильная адаптивность для слайд-презентаций
## Практические правила и решения проблем

---

## 1. Проблемы viewport на iOS Safari

### Проблема: `100vh` не равно видимой области
На iOS Safari адресная строка и тулбар занимают ~94px. CSS `100vh` включает скрытую за ними область.

```
┌─────────────────────┐
│   Status bar (47px)  │
├─────────────────────┤ ← Начало видимой области
│                     │
│   Контент (100dvh)  │ ← Видимая область = 100dvh
│                     │
├─────────────────────┤ ← Конец видимой области
│  Safari toolbar     │
│  (47px)             │
└─────────────────────┘
        ↕
   Полная высота = 100vh
```

### Решение: `100dvh` (Dynamic Viewport Height)

```jsx
// ПЛОХО — обрезается на iPhone
<div className="h-screen">

// ХОРОШО — учитывает интерфейс браузера
<div style={{ height: '100dvh' }}>
```

| Единица | Поведение | Поддержка |
|---|---|---|
| `vh` | Фиксированная, включает скрытую область за тулбаром | Все браузеры |
| `dvh` | Динамическая, учитывает текущее состояние тулбара | Safari 15.4+, Chrome 108+ |
| `svh` | Минимальная видимая область (тулбар видим) | Safari 15.4+, Chrome 108+ |
| `lvh` | Максимальная (тулбар скрыт) | Safari 15.4+, Chrome 108+ |

### Viewport meta-тег

```html
<!-- Обязательно для iPhone safe areas -->
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover" />
```

`viewport-fit=cover` позволяет использовать `env(safe-area-inset-*)` для учёта "чёлки" и закруглённых углов.

## 2. Safe Area Insets

### Нижняя панель управления на iPhone

```css
/* Отступ, учитывающий домашнюю полоску iPhone */
padding-bottom: max(0.5rem, env(safe-area-inset-bottom));
```

### Применение в проекте

```jsx
// Нижняя панель навигации (точки, PDF, fullscreen)
<div style={{ paddingBottom: 'max(0.5rem, env(safe-area-inset-bottom))' }}>
```

## 3. Overflow и прокрутка на мобильных

### Проблема: контент обрезается без возможности прокрутки

На десктопе слайд умещается в viewport. На мобильном — контент выше viewport. Если `overflow: hidden`, пользователь не увидит нижнюю часть.

### Решение: двухуровневая overflow-стратегия

```
┌── Внешний контейнер (overflow: hidden) ──┐
│  ← Блокирует скролл страницы             │
│                                           │
│  ┌── Контент слайда (overflow-y: auto) ──┐│
│  │  ← Разрешает прокрутку внутри слайда  ││
│  │                                       ││
│  └───────────────────────────────────────┘│
└───────────────────────────────────────────┘
```

```jsx
// Внешний контейнер — блокирует скролл всей страницы
<div className="relative w-full overflow-hidden" style={{ height: '100dvh' }}>

  // Внутренняя область слайда — разрешает вертикальную прокрутку
  <div className="flex-1 overflow-y-auto">
    {children}
  </div>
</div>
```

### Touch-scroll на iOS

```css
/* Инерционная прокрутка для iOS */
-webkit-overflow-scrolling: touch;
```

## 4. Tailwind: правила адаптивных размеров

### Breakpoints для мобильной версии

| Экран | Tailwind prefix | Примерные устройства |
|---|---|---|
| < 640px | (default) | iPhone, Android phones |
| ≥ 640px | `sm:` | Большие телефоны (landscape) |
| ≥ 768px | `md:` | iPad mini, планшеты |
| ≥ 1024px | `lg:` | iPad, ноутбуки |
| ≥ 1280px | `xl:` | Десктопы |

### Типографика: мобильные размеры

```
Правило: на мобильном шрифт = desktop × 0.55–0.7
```

| Роль | Мобильный | md+ | lg+ |
|---|---|---|---|
| H1 (главный заголовок) | `text-2xl` (24px) | `text-5xl` | `text-6xl`–`text-7xl` |
| H2 (секция) | `text-2xl` (24px) | `text-4xl` | `text-5xl` |
| H3 (подзаголовок) | `text-sm`–`text-base` | `text-xl` | `text-2xl` |
| Body | `text-xs`–`text-sm` | `text-lg` | `text-xl` |
| Caption | `text-[10px]`–`text-[11px]` | `text-base` | `text-lg` |
| Badge | `text-[10px]` | `text-xs` | `text-sm` |

### Отступы (padding/margin)

```
Мобильный padding = desktop × 0.4–0.6
```

| Элемент | Мобильный | md+ | lg+ |
|---|---|---|---|
| Страничный padding-x | `px-6` | `px-16` | `px-24` |
| Карточка padding | `p-2`–`p-3` | `p-5`–`p-6` | `p-8` |
| Gap между карточками | `gap-2` | `gap-3`–`gap-4` | `gap-5` |
| Margin после заголовка | `mb-1`–`mb-2` | `mb-4`–`mb-6` | `mb-8` |

### Нижний отступ контента (pb)

```jsx
// Обязательный запас внизу для навигационной панели
<div className="pb-10 lg:pb-12">
```

## 5. Grid-layouts на мобильных

### Правило: max 2 колонки на мобильном

```jsx
// ПЛОХО — 4 колонки на телефоне = нечитаемо
<div className="grid grid-cols-4">

// ХОРОШО — 2 колонки на мобильном, 4 на десктопе
<div className="grid grid-cols-2 lg:grid-cols-4 gap-3 md:gap-5">
```

### Рекомендации по grid-cols

| Десктопный layout | Мобильный layout |
|---|---|
| `lg:grid-cols-4` | `grid-cols-2` |
| `lg:grid-cols-3` | `grid-cols-1` |
| `lg:flex-row` (2 колонки) | `flex-col` |

## 6. Justify-content на мобильных

### Проблема: `justify-center` + overflow

На десктопе `justify-center` центрирует контент красиво. На мобильном, если контент больше контейнера, `justify-center` скрывает верхнюю часть (центрирует на нижнюю половину).

### Решение

```jsx
// justify-start на мобильном, justify-center на десктопе
<div className="flex flex-col justify-start lg:justify-center">
```

## 7. Чек-лист перед релизом мобильной версии

### Критические проверки

- [ ] Используется `100dvh` вместо `100vh` / `h-screen`
- [ ] Добавлен `viewport-fit=cover` в meta-тег
- [ ] Нижняя панель использует `env(safe-area-inset-bottom)`
- [ ] Контент-область имеет `overflow-y: auto`
- [ ] Нижний padding контента ≥ 40px (`pb-10`) — запас для навигации
- [ ] Шрифты уменьшены для мобильных (default < md breakpoint)
- [ ] Grid max 2 колонки на мобильном
- [ ] Padding-x ≤ 24px (`px-6`) на мобильном
- [ ] `justify-start` на мобильном, `lg:justify-center` на десктопе
- [ ] Touch-swipe навигация не блокирует вертикальную прокрутку

### Тестовые viewport'ы

| Устройство | Размер | Особенности |
|---|---|---|
| iPhone SE | 375×667 | Самый маленький актуальный экран |
| iPhone 14 | 390×844 | Стандартный iPhone |
| iPhone 14 Plus | 428×926 | Большой iPhone |
| iPhone (с Safari) | 390×**750** | Реальная видимая область (минус тулбары) |
| iPad mini | 768×1024 | Минимальный планшет |

**ВАЖНО**: Всегда тестируйте с viewport height = 750px (а не 844px) чтобы имитировать реальную видимую область iPhone с Safari.

## 8. Типичные ошибки и их решения

### Ошибка 1: `overflow: hidden` на всех уровнях
```css
/* App.css — корневые стили */
html, body, #root { overflow: hidden; }
```
**Проблема**: Блокирует любую прокрутку.
**Решение**: Оставить `overflow: hidden` только на внешнем контейнере, внутри — `overflow-y: auto`.

### Ошибка 2: `h-screen` вместо `dvh`
**Проблема**: На iOS Safari контент уходит за тулбар.
**Решение**: `style={{ height: '100dvh' }}`.

### Ошибка 3: Одинаковые шрифты для мобильного и десктопа
**Проблема**: `text-lg` (18px) body текст на 390px экране слишком крупный.
**Решение**: `text-xs md:text-lg` — мелкий на мобильном, нормальный на десктопе.

### Ошибка 4: `grid-cols-1` для карточек на мобильном
**Проблема**: Вертикальный список карточек не помещается в viewport.
**Решение**: `grid-cols-2 lg:grid-cols-4` — компактнее за счёт 2 колонок.

### Ошибка 5: Touch-swipe блокирует scroll
**Проблема**: Touch handler на родителе перехватывает все жесты.
**Решение**: Не вызывать `preventDefault()` в touch-обработчиках, фильтровать по горизонтальному threshold (60px+).
